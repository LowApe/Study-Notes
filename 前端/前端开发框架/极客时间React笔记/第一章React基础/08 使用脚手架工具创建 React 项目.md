> 脚手架：保证工程顺利进行而搭建的工作平台

1. Create React App (最小的)

为什么使用脚手架?
> 前端项目通常引入多个技术站是很复杂的
1. React **UI开发**
2. Redux 状态的管理
3. React/Router **路由管理**
4. BABEL **新的 JavaScript 特性翻译游览器旧的代码**
5. webpack **打包**
6. ESLint **语法检查**
